4️⃣ CONTRACT CREDIT SYSTEM
------------------------------------

Replace coins with **Contract Credits**

Backend rules:
- Default credits on signup = 3
- 1 Contract creation = 1 credit consumed

Conditions:
- If credits > 0 → allow contract creation
- If credits = 0 → redirect to Pricing page

Show remaining credits on:
- Dashboard
- Profile page

------------------------------------
6️⃣ PAYMENT GATEWAY – PAYU (IMPORTANT)
------------------------------------

Use **PayU Payment Gateway (Redirect Checkout)**

Payment flow:
1. User clicks “Buy Contract Credit”
2. Backend creates PayU payment request
3. Generate required PayU hash on backend
4. Redirect user to PayU hosted payment page
5. On successful payment:
   - PayU redirects to success URL
   - Verify payment response securely
   - Increase Contract Credits by +1
6. On failure:
   - Redirect to failure page
   - Show error message

Security rules:
- Hash generation MUST be server-side
- Never expose PayU key/secret to frontend
- Validate payment status before updating credits
